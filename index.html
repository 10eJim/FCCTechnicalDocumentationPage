Introduction

A-Frame is a web framework for building virtual reality (VR) experiences. A-Frame was developed to be an easy but powerful way to develop VR content. As an independent open source project, A-Frame has grown to be one of the largest and most welcoming VR communities.

A-Frame is built on a three.js framework, providing users access to:

  • Javascript
  • DOM APIs
  • three.js
  • WebVR
  • WebGL

    • Since A-Frame allows for simple HTML and Javascript manipulation, anyone familiar with both can quickly develop impressive WebVr experiences that run completely in the browser. This documentation is intended to introduce you to the environment component, which allows you to generate a series of fantastic environments for your VR area. As I have been working with simple interactions to learn about using A-Frame, I have found placing them in an environment makes those simple experiences much more satisfying.

    What you should already know

    This guide assumes you have the following basic background:

  • A general understanding of the Internet and the World Wide Web (WWW).
  • Good working knowledge of HyperText Markup Language (HTML).
  • Some programming experience. If you are completely new to programming, I would suggest starting elsewhere.
    • Initial A-Frame Set-up

    Initializing our A-Frame page is simple and easy. First, we need to include the A-Frame script tag in the head and our a-scene tag in the body. All of our A-Frame tags will go inside the A-scene tag. Like this:

    <html>
    <head>
    <script src="https://aframe.io/releases/0.8.0/aframe.min.js"></script>
    </head>
    <body>
    <a-scene>
    </a-scene>
    </body>
    </html>

    This initial set-up will set up WebGL, the canvas, camera, lights, renderer, render loop and visual inspector development tools. It also implements out of the box WebVR support on platforms such as HTC Vive, Oculus Rift, Samsung GearVR, and smartphones (Google Cardboard).

    Including the Environment

    Now that we have our inital A-Frame Scene code in place, we can render an environment very easily using Diego F. Goberna's environment component, available on github. First, we simply need to add the environmental component script to our head section with our A-Frame script tag. The script tag is as follows:

    <script src="https://rawgit.com/feiss/aframe-environment-component/master/dist/aframe-environment-component.min.js"></script>

    Now all you have to do is add an environment component inside your a-scene tag. The environment component looks like this:

    <a-entity environment></a-entity>

    You have now included the default environment component to your scene. In the next section, we'll look at the other environments we can use and how to alter them from their default settings.

    Selecting an Environment

    The component includes a number of preset environments in it's "preset" attribute. You can easily select from the preset environments by including your selection in the code as follows:

    <a-entity environment="preset: (name of the preset)"></a-entity>

    The component currently offers presets of default, contact, egypt, checkerboard, forest, goaland, yavapai, goldmine, threetowers, poison, arches, tron, japan, dream, volcano, starry, and osiris. So, if we wanted to select the forest, for example, we would specify as follows:

    <a-entity environment="preset: forest"></a-entity>

    Great. Now we can select the environment we want to include. To preview the available options, take a look at aframe-environment-component Test Page

    Adjusting the Parameters

    As with all A-Frame entities, the enviroment component has attributes that can be altered to your preferences by editing them from their default values.

    The easiest way to do this is to open the visual inspector, while examining your scene, using "ctrl-alt-i" on your keyboard to launch the visual inspector. Search for the "environment" tab on the left side filter and select it. You can then alter the default attributes on the right, until the scene is to you liking. You can then press the "Copy Attributes" button to clipboard your changes and put them in your code. You alter the values of the default settings by including your attribute selections after preset, seperated by semi-colons. Like this:

    <a-entity environment="preset: forest; groundColor: #445; grid: cross"></a-entity>

    For a complete chart of all of the attributes that can be altered, refer to the component's Github Repository
